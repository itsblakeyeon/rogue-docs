# 문서 관리

## 폴더 구조

```
rogue-docs/
├── 1-목표/        — 목표, 방향성, 우선순위
├── 2-실행/        — 주간 실행, 액션 아이템
├── 3-미팅/        — 모든 미팅 기록 (프로젝트 미팅 포함)
├── 4-프로젝트/    — 프로젝트별 폴더 (문서 + 코드)
│   ├── 병원마케팅/
│   ├── 마오타이/
│   └── 업무환경구축/
├── 5-자료/        — 첨부 자료 (이미지, PDF, 가이드)
└── 0-시스템/      — 팀 운영 시스템 규칙 (이 폴더)
    └── 템플릿/    — 문서 템플릿
```

## 문서 위치 기준

- 목표, 방향성 → `1-목표/`
- 주간 실행, 액션 → `2-실행/`
- 미팅 기록 → `3-미팅/` (파일명: `날짜-프로젝트-주제.md`)
- 프로젝트 소속 → `4-프로젝트/{프로젝트명}/`
- 첨부 파일, 참고 자료 → `5-자료/`

## 프로젝트 문서 규칙

- 각 프로젝트 폴더에는 **`개요.md`**를 마스터 문서로 둔다 (목표, 전략, 계획, 현황, 회고)
- 프로젝트 내부는 **`문서/`**와 **`코드/`**로 구분한다
  - `문서/` — 기획, 리서치, 제안서 등 사람이 읽는 파일
  - `코드/` — 스크립트, 크롤러, 웹사이트 등 실행되는 파일
- `개요.md`와 `CLAUDE.md` 등 시스템 파일은 프로젝트 루트에 둔다

## 작성 규칙

- 사람이 읽는 문서는 **한국어 파일명** (`개요.md`, `기획안.md` 등)
- 시스템/도구용 파일은 영문 허용 (`CLAUDE.md`, `.gitignore` 등)
- 폴더명, 내용 모두 **한국어** (코드/기술 용어는 영문 허용)
- 파일명 띄어쓰기 대신 하이픈(`-`) 사용: `마케팅-기획안.md`
- 커밋 메시지는 한국어
- 민감 정보(API 키 등)는 절대 커밋하지 않음

## 문서 간 링크 규칙

- 문서 간 참조가 가능하면 **항상 상대 경로 링크**를 건다
- 형식: `[표시 텍스트](상대/경로/파일.md)`
- 예: `[시스템/문서-관리.md](../../시스템/문서-관리.md)`
- 링크가 깨지면 즉시 수정한다

## 리스트 우선 규칙

- 마크다운 테이블 대신 **리스트를 기본**으로 사용한다
- 테이블은 열이 3개 이상이고 비교가 필요한 경우에만 허용한다
- 목표, 전략 등 서술형 섹션도 **불릿 리스트**로 정리한다 (줄글 금지)
- 이유: 마크다운 테이블은 편집이 어렵고 diff가 깨지기 쉬움, 줄글은 스캔이 어려움
