# 운영 관리

## 구조

```
운영/
├── 목표/     — OKR 파일 (분기별)
├── 실행/     — 주간 실행 파일
└── 기타 운영 문서
```

## OKR 규칙

- **주기**: 바이위클리 (2주 단위)
- **구성**: Objective 1개, Key Result 2~3개
- **파일**: 분기별 1개 파일에 누적 기록
  - 파일명: `YYYY-QN.md` (예: `2026-Q1.md`)
  - 새 바이위클리가 시작되면 같은 파일에 섹션 추가
- **Objective**: 정성적, 방향성 중심 (측정 불가능해도 됨)
- **Key Result**: 정량적, 측정 가능, 2주 안에 달성 가능한 수준
- **위치**: `운영/목표/`

## 주간 실행 규칙

- **주기**: 1주 단위 (월요일 시작)
- **파일명**: `YYYY-WNN.md` (예: `2026-W12.md`)
- **구조**: 예정 → 완료 → 회고
- **프로젝트 할일 연동**:
  - 할일의 원본은 각 프로젝트의 `개요.md` 계획 섹션
  - 주간 실행 파일에는 해당 주에 할 일만 링크로 가져온다
  - 예: `- [ ] [병원마케팅 제안서 발송](../../프로젝트/병원마케팅/개요.md)`
- **위치**: `운영/실행/`

## 운영 루틴

- **월요일**: 주간 실행 파일 생성, 이번 주 할 일 정리
- **금요일**: 완료 항목 체크, 회고 작성
- **격주 월요일**: 새 바이위클리 OKR 설정 (이전 OKR 결과 정리 후)
