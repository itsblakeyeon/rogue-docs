# 문서 변경 Slack 알림 — 설계

## 목적

`main` 브랜치에 push 시, `4-프로젝트/` 하위 `.md` 파일 변경을 감지하여 `#1-do` 채널의 프로젝트별 스레드에 알림을 전송한다.

## 결정 사항

- **채널**: `#1-do` (C0AM5077A0Z)
- **스레드 매핑**: 프로젝트당 스레드 하나, ts를 리포 내 JSON 파일로 관리
- **트리거**: `main` push마다
- **대상**: `4-프로젝트/` 하위 `.md` 파일 변경만 (비프로젝트 문서 무시)
- **알림 내용**: 변경 파일 목록 + 커밋 메시지 (AI 요약 없음)
- **시크릿**: `SLACK_BOT_TOKEN`만 필요

## 구조

```
.github/
├── workflows/
│   └── doc-notify.yml          # GitHub Actions 워크플로우
├── scripts/
│   └── doc-notify.mjs          # 알림 로직
└── slack-threads.json           # 프로젝트명 → 스레드 ts 매핑
```

## 동작 흐름

1. `main` push 트리거
2. `git diff`로 변경된 파일 목록 추출
3. `4-프로젝트/{프로젝트명}/` 경로의 `.md` 파일만 필터
4. 프로젝트별로 그룹핑
5. 각 프로젝트에 대해:
   - `slack-threads.json`에서 스레드 ts 조회
   - ts 없으면 → `#1-do`에 새 메시지 생성 (스레드 시작), ts 저장
   - ts 있으면 → 해당 스레드에 reply
6. `slack-threads.json`에 변경 있으면 커밋+push

## 메시지 포맷

### 새 스레드 (첫 알림)
```
📁 [병원마케팅] 문서 변경 알림
```

### 스레드 reply
```
📝 문서 변경 (2026-03-21)

변경 파일:
• 개요.md
• 문서/제안서-초안.md

커밋: feat: 제안서 초안 작성
```

## 성공 기준

- push 후 1~2분 내 `#1-do`에 프로젝트별 스레드로 알림 도착
- 같은 프로젝트의 후속 알림은 같은 스레드에 reply로 달림
