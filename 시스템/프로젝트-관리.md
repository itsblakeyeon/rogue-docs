# 프로젝트 관리

## 프로젝트 생성 규칙

새 프로젝트를 만들 때:

1. `프로젝트/{프로젝트명}/` 폴더 생성
2. `개요.md` 생성 — [템플릿](템플릿/프로젝트-개요.md) 사용
3. 필요에 따라 `문서/`, `코드/` 하위 폴더 생성
4. [프로젝트/현황.md](../프로젝트/현황.md)에 프로젝트 추가

## 프로젝트 폴더 구조

```
프로젝트/{프로젝트명}/
├── 개요.md      — 마스터 문서 (목표, 전략, 계획, 현황, 회고)
├── CLAUDE.md    — Claude Code 컨텍스트 (필요 시)
├── 문서/        — 기획, 리서치, 제안서 등
└── 코드/        — 스크립트, 웹사이트 등
```

## 개요.md 구조

모든 프로젝트의 `개요.md`는 다음 섹션 순서를 따른다:

- **목표** — 무엇을 왜 하는지
- **전략** — 어떻게 달성할지, 방향과 접근법
- **계획** — 구체적 작업 항목 (체크리스트)
- **현황** — 지금 어디까지 왔는지
- **회고** — 완료 후 결과, 배운 점

## 할일 관리

- 프로젝트 할일의 **원본**은 해당 프로젝트의 `개요.md` 계획 섹션
- 주간 실행 파일(`운영/실행/`)에서는 이번 주에 할 일만 링크로 참조
- 할일 완료 시 `개요.md`의 체크박스를 `[x]`로 변경

## 프로젝트 현황

- [프로젝트/현황.md](../프로젝트/현황.md)에서 전체 프로젝트 목록과 상태를 한눈에 확인
- 프로젝트 상태: 진행중 / 대기 / 완료
