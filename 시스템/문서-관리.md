# 문서 관리

## 폴더 구조

```
rogue-docs/
├── 시스템/        — 팀 운영 시스템 (이 폴더)
├── 운영/          — 조직 운영 문서 (가이드, 정책)
├── 리서치/        — 시장 분석, 조사
├── 자료/          — 첨부 자료 (이미지, PDF)
├── 미팅/          — 모든 미팅 기록 (프로젝트 미팅 포함)
├── 팀/            — 팀 관련 문서 (목표, 방향성)
└── 프로젝트/      — 프로젝트별 폴더 (문서 + 코드)
    ├── 병원마케팅/
    ├── 마오타이/
    └── 업무환경구축/
```

## 문서 위치 기준

| 문서 성격 | 위치 |
|----------|------|
| 프로젝트 소속 | `프로젝트/{프로젝트명}/` |
| 미팅 기록 | `미팅/` (파일명: `날짜-프로젝트-주제.md`) |
| 범용 리서치 | `리서치/` |
| 조직 운영 | `운영/` |
| 팀 관련 | `팀/` |
| 첨부 파일 | `자료/` |

## 프로젝트 문서 규칙

- 각 프로젝트 폴더에는 **`개요.md`**를 마스터 문서로 둔다 (목표, 배경, 계획, 현황)

## 작성 규칙

- 사람이 읽는 문서는 **한국어 파일명** (`개요.md`, `기획안.md` 등)
- 시스템/도구용 파일은 영문 허용 (`CLAUDE.md`, `.gitignore` 등)
- 폴더명, 내용 모두 **한국어** (코드/기술 용어는 영문 허용)
- 파일명 띄어쓰기 대신 하이픈(`-`) 사용: `마케팅-기획안.md`
- 커밋 메시지는 한국어
- 민감 정보(API 키 등)는 절대 커밋하지 않음
